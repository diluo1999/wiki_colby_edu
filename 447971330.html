Di Luo : Di Luo's CS232 Project 2

Created by Di Luo, last modified on Sep 23, 2019

Description

In this project, we continued working on Quartus and implement circuits in both VHDL and graphic design. We learned how to use the built-in templates of VHDL code to create new units in the circuit and how to use DE1 hardware board to test the validity of the circuit. This project has two tasks. The first task is to implement a 7-segment display driver that takes in four bits and drives a 7 segment display for the hexadecimal characters 0-9, A-F. The second task is to implement a circuit that takes in two 4-bit binary numbers and outputs their sum as a 5-bit binary number. After implementation, the circuits were tested with the 7-segment display on the DE1 board and the validity of both circuits were proved.

 

Task 1

For task 1, we need to implement a 7-segment display driver that takes in four bits and drives a 7 segment display for the hexadecimal characters, and test it with DE1 board. I decided to implement the driver with VHDL code. Then in order to output the correct patterns on the 7-segment display, I created a truth table showing the segments that needs to light up for each hexadecimal number in 0-9-A-F. Additionally the index of the each segment is shown in the picture aside.

Then I created VHDL file driver.vhd to implement the driver, in which the code guides the 7-segment display to show the right pattern when each hexadecimal number is inputed. Since the default state of each segment is lighted up when the input is all zero, the input of 1 would make the light of the segment off.

Demonstrate the 7-segment driver circuit using an 8-bit counter that drives two seven segment displays. This will require you to include two of your 7-segment display drivers into your top level design. Download this circuit onto the board and test it. To see the numbers, you may want to use a 24-bit counter and use the highest 8-bits as the input to your 7-segment driver.

Eventually in order to demonstrate the validity of the 7-segment driver circuit, I created hexDisplay.bdf to implement the top-level design that has an 8-bit counter that drives two 7-segment displays. The input are a clock and a reset, which control the counter to input an 8-bit number or reset. Then the first 4 bits of the 8-bits number work as the input of a display driver and the other 4 bits work for another display driver. Each driver sends out an 7 outputs to control a display. The graphic design of hexDisplay is shown below.

Then the circuit was downloaded onto the board for testing. I tested all the possible numbers from 00 to FF, and a portion of the testing process is shown in the video below. The correct outputs proved the correctness of the circuit.

Extension

Extension 1&2: Add functionality to your second circuit so it can execute operations other than addition & implement a minus sign to show negative numbers

For extension 1&2 I decided to improve the second circuit AddIt by adding the functionality of subtraction to the circuit. I continued working in the project of Task 2 (addIt). First I created a new VHDL file addorsub.vhd which is similar to adder.vhd but has one more input "sub" and one more output "minus". If "sub" equals to 0, the circuit will do addition; if "sub" equals to 1, then the circuit will do subtraction. For the output "minus", it indicates whether the output is a negative number. If "minus" equals to 0, the third 7-segment display will display nothing; if "minus" equals to 1, then the third 7-segment display will display a minus sign meaning that the output number is negative.

When I tried to complete the VHDL file of addorsub, I came across a lot of problems in the IF-THEN-ELSE statements, which works differently from Python or Java. Extension 3: 2's complement numbers

For extension 3 I changed VHDL files and BDF file in task 2 in order to make the addIt circuit work for 2's complement numbers. Since the input is 4-bit number, the most significant digit of a 4-bit 2's complement number represents whether the number is negative, if 0 then positive and if 1 then negative. This also means that a 4-bit 2's complement number can only represent the number from -8 to 7, and the sum of two numbers ranges from -16 to 14. The truth table for a 4-bit 2's complement number is shown below. We can find that the truth table is the same as the one for normal unsigned number, but the addition process will be different. 

Then I added a new input twoscomp into addorsub.vhd, which controls whether the input numbers are 2's complement numbers. If it's in 2's complement, twoscomp = 1, and if not twoscomp = 0. The refined code is stored in addorsub(twoscomp).vhd and is shown below. The addition of 4-bits 2's complement numbers is by the formula c <= (a(3) & a) + (b(3) & b). I received help from Yixuan Qiu on finding the formula.

Then I renewed the addIt.bdf, which is stored in addIt(twoscomp).bdf, and used SW4 on board to control twoscomp. The testing video is shown below, which proves the validity of the circuit since the second display (HEX1) works correctly as the most significant digit of 2's complement number. If the number is negative, HEX1 display 1; if positive, HEX1 display 0.

Reflection
