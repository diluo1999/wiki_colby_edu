Di Luo : Di Luo's CS151 Project 8

Created by Di Luo, last modified on Nov 06, 2018

Description

In Project 8, we continued working on turtle and L-system to make a scene that consists of fractal trees, but we rewrote our functions with Object-oriented design (OO). A L-system is a parallel rewrite system that can be used to make complex strings, including a simple base string and a rule to replace the characters in the base string. Object-oriented design is "the process of planning a system of interacting objects for the purpose of solving a software problemIn our project" and to be more specific we converted both the lsystem and the turtle interpreter modules into classes. Besides, we also enabled our functions to handle multi-rule L-systems in order to draw more complex shapes. 

In our project, we firstly modified our drawString method so that it could handle some new characters in the strings. Next in arrangement.py we created a function that could make an arrangement scene of trees. Finally in growth.py we created an image of a scene of trees using variation L-systems after 2, 3, and 4 iterations.

Extensions

For extensions, firstly, I created a method __str__(self) in lsystem.py which returns a formatted string of base and rules of L-system. "The __str__ method gets called automatically when an object is printed out. It's also called to implement casting an object to a string using str(thing)." Then I rewrote the test function in lsystem.py and successfully printed out the base and rules from a file.

Then I created the function text(self, object) in turtle_interpreter.py which prints out the base and rules of L-system (object) in a dialog.

Finally I created text.py, which draws a tree from a L-system and also prints out the base and rules in a dialog.

 

For the second extension, I created two more shapes in text.py by passing strings to the turtle interpreter's drawString method. One is a colorful square and the other is a colorful dumbbell. The string of dumbbell is shown below. This method is also used in scene.py.

 

For the third extension, in Required Image 1 the trees in the arrangement have vary colors of branch, vary numbers of iteration and vary angles.

For the fourth extension, I added many new symbols to turtle.interpreter.py and applied them to the L-system shown in the scene.py.

For the fifth extension, two variant L-systems were created in system3.txt and system4.txt.

For the sixth extension, I created scene.py which draws a scene including a road, some new L-system trees with decorations of leaves and berries, a house combined by two L-systems, sky and ground.

 

Reflection

In project 8, I reviewed the knowledge in turtle, L-system and classes that we learned in class. I modified lsystem.py and turtle_interpreter.py with Object-oriented design, in which both modules were transferred into classes. I also learned how to draw more complex geometric shapes by adding more new characters into both L-system and its interpreter document turtle_interpreter.py, such as 'L', which draws a leaf. Besides, I learned how to rewrite a L-system with more characters in order to create more complex shapes.
