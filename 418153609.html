Di Luo : Di Luo's CS151 Project 11: 3D Scenes

Created by Di Luo, last modified on Dec 06, 2018

Description

In Project 11, we extended our drawing system from standard 2D turtle to 3D turtle, by which we can make 3D shapes as well as 2D shapes. In the project we need to demonstrate the use of the 3D turtle and build some interesting 3D shapes, and also pick one significant extension of the system and design and implement our own solutions. To demonstrate the use of the 3D turtle, I created many new shapes such as Cylinder and Ball, which can create 3D shapes, and I made a 3D scenes of Miller Library that incorporates my shapes. Regarding making my own task and solution, 


Task 1

Firstly, I created four new 3D shape classes, which are Cuboid, Cylinder, Ball, and TriangularPrism. All of these classes are the subclasses of Shape and can create a string. The shapes can be drawn by using Turtle interpreter. Then I created a test function in shapes.py to show my new 3D shapes. An image with examples of my four new 3D shapes is below. In this image, there are four groups of shapes. The black group has two cuboids, one on the left is normal and the other has jitter style. The red group has two cylinders, one on the left is normal and the other has dotted style. The green group has two ball that are composed by three circles in three dimensions. The blue group has two triangular prisms, one on the left is normal and the other has dotted style.

Task 2

Secondly, I made a 3D scene of the Miller Library that incorporates my shapes. The scene is realistic style. In the process of design, I designed two cuboids as the base of the library, with the bigger one on the bottom and the smaller one on the top. Then fours cylinders and a tall cuboid are put on the base as the pillars and main building. Next, a triangular prism is set as the roof of the library. On the top of the roof, a cylinder, a ball and a thin cuboid combined with each other as the the bell tower of the library.

Task 3

Then I was thinking about if I can make the roof look better by showing the inner structure of the roof. In order to do that I need to use the recursion that we learned in this week's class to create the binary tree of triangular prisms. The binary tree is from the top of the roof to the bottom and each prism gives rise to up to 2 below it. The reason why we use recursion is that the whole stack of prisms is a binary tree, and left and right stack of prisms beneath the top prism are also binary trees. So firstly I added the function drawBinaryTriangularPrism to TriangularPrism class, which has x, y, z as parameters. To get the base case (or stopping condition) for recursion, I added the parameter nLevels and the if statement of base case to the function.

In order to avoid creating another object of TriangularPrism to draw the binary tree, I added the fields w, d and the functions setw and setd to the TriangularPrism class so that I can change the value of w and d. Then to renew the drawstring I created function resetString to renew the string with new w and d. Extensions

Extension 1: Design multiple 3D, parameterized L-systems

For the first extension I created two more L-systems in systemZ2.txt and Lsystem.txt. The L-system in systemZ2.txt was the variation of the one in systemZ2.txt given in the lab. The structure of the tree was changed to be more stereoscopic by adding more branches in different directions and adding replacements to different rules. Decorations were also added.

Extension 2: Pick some semi-complex shapes and create efficient designs for them

For the second extension, I created a wire frame geodesic sphere that is composed by many circles in different dimension. The shape can be seen by running 3DScene2.py, which is for my new scene, run the code inside, and right click on the window after all other shapes show up.

 

Extension 3: Make an additional scene

For the third extension, I created a new scene of Miller library in 3DScene2.py with the new roof, new trees on the two sides of the library, and also a wire frame geodesic sphere in the middle (need the right click for showing up).

 

Extension 4: Set right mouse call back

For the fourth extension I tried to use the function setRightMouseCallback to make the scene more interesting. Firstly, I added the function setRightMouseCallback in TurtleInterpreter class, which calls the function setRightMouseCallback in the module turtleTk3D. The setRightMouseCallback function has one parameter – function, which is the function that needs to be run after the right click on the window. By doing this I can call the setRightMouseCallback function in other files without import turtleTk3D.

Secondly, I defined the function Sphere in the test function, which has the code for drawing the wire frame geodesic sphere.

Thirdly, I added the code that calls the function setRightMouseCallback with the parameter Sphere at the end of the test function, but before the function hold.

Finally, I ran the code in 3DScene2.py again. Without the right click, the scene would be the same as what is shown above. But when the user right clicks on the window, the wire frame geodesic sphere will shows up.

 

Reflection

In Project 11, I learned how to employ the 3D turtle to create 3D shapes and scenes. I also studied how to make my own task and to find efficient and elegant solution for the task. By demonstrating how I define the task, design a solution, and implement the solution so the computer can complete the task, I became clearer with the whole process.
