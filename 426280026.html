Di Luo : Di Luo's CS231 Project 5: Decision-making Simulation: Checkout Lines

Created by Di Luo, last modified on Mar 26, 2019

Description

In Project 5 I worked on using my Queue implementation from lab to simulate shoppers picking which check-out line to use. The queue is used to insert elements at the end of the queue and removes from the beginning of the queue, which is like a check-out line in supermarkets. I also implemented and investigated the effectiveness of three strategies for choosing a queue. The first strategy is to randomly choose a queue; the second one is to check each of the queues and choose the one with the least customers; the third one is to randomly choose two queues, compare them, and choose the one with fewer customers. By showing the simulation of each strategy using LandscapeDisplay, I created a simulation of the process and it enabled me to analyze the performance of each strategy.

 

Solution

Firstly I created the Customer class, which represent the customer with items that need to be paid and a counter recording how many time-steps it takes for them to choose a line and paid for all of the items, and CheckoutAgent class, which maintains a queue of Customers and conducts the money collection process. The CheckoutAgent is displayed graphically as a tall rectangle and its height represents the number of customers in the queue.


Then I created the Landscape class to show the scene of the checkout lines, in which I created an ArrayList<CheckoutAgent> to hold the list of checkout agents, and a LinkedList<Customer> that I created in the last project to hold the Customers who have left the queues. It also has a function that computes and prints the average and standard deviation of the time-to-leave for all of the Customers in the finished customer list, which is shown below:

.

After that, I used the LandscapeDisplay file to display the scene of the check-out line, which is shown below:


Next, I created RandomCustomer, Pick2Customer, and PickyCustomer, which are respectively the customer with a different strategy. 

 

  • RandomCustomer extends the Customer class and implements the random line-choosing strategy.
  • Pick2Customer also extends the Customer class and implements the strategy in which the customer randomly chooses two lines, then joins the shorter of those two.
  • PickyCustomer also extends the Customer class and implements the strategy in which the customer chooses to join the shortest queue.

Then I created RandomCustomerSimulation, Pick2CustomerSimulation, PickyCustomerSimulation, which are the main programs that run simulations with the given type of Customer. An animated gif of the RandomCustomerSimulation is shown below:

The test codes in each simulation that I adapted for my simulation from TestRandomCustomer.java give each Customer 1 to 10 items.

The lines grow over time so that each line becomes long as the number of simulation increases, meaning that 1 to 10 items is too many. Then I adjust the maximum number of items to find a situation in which the queues don't grow over time. Firstly I changed it to 1 to 50 items and the lines grow too long with too many items per Customer:

(100 times simulation)

Then I changed it to 1 to 6 items and then it shows short lines with this new manageable number of items per Customer: 

(100 times simulation)

Then I continue doing my project with the new, smaller number from 1 to 6. Finally by running each simulation the printFinishedCustomerStatistics() function in Landscape class prints out the the average and standard deviation of the time-to-leave for all of the Customers in the finished customer list.  Each simulation runs with lots of customers, which is 1000 in my case, and print out the statistics once every 100 time steps.

As we can see from the results, regarding average time to finish, Pick2Customer is the fastest; RandomCustomer and PickyCustomer spend the same average time. Regarding standard deviation, PickyCustomer has the lowest standard deviation, meaning that each customer spends almost the same time. Pick2Customer's standard deviation is a little higher, and RandomCustomer has a high standard deviation, meaning that the time each customer spends varies greatly. In conclusion, Pick2Customer stands out to be the best strategy since it costs the least time with a good standard deviation.


Reflection

In this project, I learned how to implement a queue to represent a check-out line in real life. I also learned how to be the most efficient customer in the supermarket, which is to randomly choose two lines and go for the shorter one. This is the first time I find that Computer Science can be really useful to analyze the real-life situation and to make our life more efficient.
