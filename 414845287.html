Di Luo : Di Luo's CS151 Project 4

Created by Di Luo, last modified on Sep 30, 2018

Description:

In Project 4, we need to make a collection of images in the style of Andy Warhol by manipulating the pixel colors of an image. In order to do so we need to apply our knowledge in John Zelle's graphics package and make use of the knowledge we learned before, such as parameters, loops, and conditional statements. For this assignment I created two python programs, One is to generate a Warhol style collage and the other is to change the green screen to a different color. Both of the programs result to image files that can be viewed with the show program.

 

Task 1
First I created a function placeImage, which contains 4 parameters, to place a source image into a destination image. The first argument is a destination Image, the second argument isrce Image, and the last two arguments are the (x, y) location to place the second Image into the first. Task 2
Then I created three more functions besides swapRedBlue(), which are grayScale(), setBlueAs0(), darker(), to edit the colors in a Image to achieve some effect. swapRedBlue() exchanges the red channel of each pixel with the blue channel for a image; grayScale() converts each pixel from color to an appropriate shade of gray;  setBlueAs0() changes the blue channel of each pixel to 0; and darker() changes the red, green, blue channels of each pixel by half to make the image darker.

Task 3
Then I wrote my main Warhol program in warhol.py, which read in one image: miller.ppm, created four copies of it using clone(), used my four filter functions to alter them, created a new blank image that is 2*width by 2*height and then used the placeImage function to insert my four edited images. Finally, it wrote out the collage image and was saved to collage.ppm shown below:

Task 4
Finally, I wrote a python program bluescreen.py that read in my green-screen image and turned the background pixels to blue with the rest of the pixels remained untouched. I used nested for loops to check each pixel in the image and test if it was very green. If so, the program changed its color to blue; If not, it remained its original color. In my test, if the green channel is 2.3 times the red channel and also bigger than the blue channel, the pixel was categorized as "very green". I put the name of the original as an argument and ran the program from command line.
(Required picture 2)

Extensions
For extensions, firstly, I created the filter grayScale(), which can convert each pixel from color to an appropriate shade of gray and in the end create a black and white version of the original image.  Gray pixels were created by "setting its red, green, and blue components to have the same brightness."  I used a weighted average of the original rgb values to determine the brightness of the gray. The data of the weight was gained from a website.
Secondly, I wrote a program to place myself in a scene in bluescreen.py. The background scene was the same size image as my green-screen image. The function placeImageWithoutBlue copied only non-blue pixels (if g < 2.3*r) from my images into the background. Then I ran the program from command line with the two arguments.

Thirdly, I created a more complex Warhol style collage in warhol.py. 

Finally, I enabled my Warhol program to read in multiple images from the command line and made a collage that integrates several images.

Reflection
In this project, I reviewed the knowledge I learned in class and lab, such as Zelle graphics package, parameters, loops, and conditional statements. I learned how to generate a Warhol style collage, how to place a source image or many source images on a destination image, how to control the color of every pixel of an image by using nested for loops, how to enable Warhol program to read in many images from the command line and made a collage, how to create a filter to generally change an image, and how to put a number of pixels that fulfill the requirements into another image instead of using the whole image.
