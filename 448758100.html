Di Luo : Di Luo's CS232 Project 3

Created by Di Luo, last modified on Sep 28, 2019

Description

This project explores sequential digital circuits by creating a reaction timer game that tests the user's reaction time. To implement the circuit, I created a 7-segment display in driver driver.vhd, a timer state machine in timer.vhd, and a top-level graphic design file in reaction.bdf. The circuit works as follows: "It begins in an idle state with two digits displaying zeros. When the user hits the start button, the circuit should light up a red LED and wait for a period of time. Then the circuit should display a green LED and start counting (approximate milliseconds), showing the count on the two digits. When the user hits the reaction button, the count should stop and freeze. If the user hits the reaction button before the green LED lights up, the counter should show FF and the system should go back to the idle state. When the user hits the start button, the digit display should be set back to zeros." (Credit to Project 3 instruction) After implementation, I tested the circuit on the board and the result shows the validity of the circuit.

 

Solution

According to the circuit description in the project instruction, I created the state machine diagram for the reaction circuit as a Moore state machine, which is shown in the picture below.

Then I created the VHDL file timer.vhd according to the state machine diagram. The inputs are clock, reset, start, and react; the outputs are mstime, which is an 8 digits binary number, redLED, and greenLED. The architecture part was implemented according to the logic in the diagram. Different states, including sIdle, sWait and sCount, return different output. In order to wait and count, I created an internal counter count, which is a 28-digit binary number. When the start is pushed, the circuit goes into sWait and the count is set to be a considerably big number in 28-digit. Then the count starts to increase by one each time until it overflows to all zeros, in which the circuit goes to state sCount and start counting with count.


Next step, I created the top-level design in reaction.bdf, which is shown in the picture below. Two drivers are used to control the two 7-segment displays.

Extension 5: Make it a game for two players, with two reaction buttons, and see who is faster

I created a new project and changed timer.vhd to timerII.vhd and reaction.bdf to reactionII.bdf to enable the two-player mode. I added one more react button so that there are reactOne (the rightmost pushbutton) and reactTwo (the leftmost pushbutton). In timerII.vhd I also added two output winLeft and winRight, which control different LEDs to indicate who is the winner. mstime is splitted into mstimeOne and mstimeTwo. In addition to original internal counter count, I added countOne and countTwo which record the reaction time of different players. More if-else statements are added in state sWait and sCount. In sWait state, if one of the players pushed react button, then the other player would win since the former player breaked the rule. In sCount state, the player who pushes the button faster would win.

Reflection

In this project, l learned how to implement a Moore state machine that has the interaction between the machine and the human. The process of creating the state machine diagram also practices my skills in understanding the description and translating the logic to a diagram. In addition, the project strengthened my ability to implement the circuit with both the VHDL file and block design.
