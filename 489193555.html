Parallel Programming in JavaScript

Task 1: Discussion on how parallelism is supported in JS

JavaScript was considered as a single-threaded programming language that ran in a browser and does not support multi-threading so that only one set of instructions is executed at any time in the same process. For instance, Google Chrome will not let a single web page’s JavaScript run concurrently because this might cause massive concurrency issues in existing web pages. All Chrome does is separate multiple components (different tabs, plug-ins, etcetera) into separate processes, but there is not a page with more than one JavaScript thread. In addition, single-threaded makes things easier for implementation and for developers using the language. JavaScript was initially a language only useful for adding some interaction to webpages, form validations, and so on — nothing that required the complexity of multithreading.

 

As CS is developing, JS starts to support muilti-threading. Some libraries of JavaScript enable JS to have parallelism, such as Parallel.js, which is a tiny library for multi-core processing in Javascript. It was created to take full advantage of web-workers API.

Task 2: Parallelism in Python – Write a function that sorts an array with parallelism

In order to create each thread, we extended threading.Thread class with our sortThread class. The run method in the sortThread class is the method that will be run in parallel threads. In this method, we used the built-in sort method in Python to sort partial array specified by the indices.

 

To make the thread working in parallel, we first create objects of the thread class. And then we call start() and join() method of thread class to make them work

For threading, with each number of threads, we created the same number of objects in sortThread class. Then we used start() and join() method of thread class to achieve multi-threading.

 

Then I tested with different array size. First is array size 1000, in which the result below shows no improvements by using more threads.

 

The result below is for array size 100000, which shows that multi-threading does improve the performance. 2-thread uses 20% time of 1-thread. 4-thread uses 30% time of 2-thread.
