Di Luo : Di Luo's CS231 Project 4: Agent-Based Simulations

Created by Di Luo, last modified on Mar 13, 2019

Description

In the project I used my linked list, which was created in lab, within the context of an agent-based simulation. Linked List is a linear data structure that its elements are not stored at the contiguous location. Instead the elements are linked using pointers. The linked list contains nodes, which contain each agent we are doing the simulation on. An agent-based simulation is a simulation that updates the current status based on each object’s condition. To do the simulation I created Agent class that creates an agent, SocialAgent class that extends Agent class and creates a group of agents, Landscape class that post the simulation, and SocialAgentSimulation class that do the simulation. Besides, I implements the updatedState() function in SocialAgent class and updateAgent() function in Landscape class to update each agents. After that I created CategorizedSocialAgent and its simulation to see how the simulation works with agents in different categories. The program ended up working as I expected, with the successful simulation according to the rules provided in the description of this project. 


Solution

Firstly I created a Agent class, which creates an agent that stores its position. Since the Agent class is going to be inherited by two more class I implemented updateState() and draw() with nothing inside as abstract functions.

 

Then I created SocialAgent class that extends Agent class. super() calls the constructor in Agent class and SocialAgent class has a draw method that draws a circle on Landscape representing the agent. If the agent moved during the last updateState it will has a lighter color, if not it will has a darker color. 

 

Then I created Landscape class that can present the agent-based simulation. In this class there is a getNeighbors() function that returns a list of the Agents within radius distance of the location x0, y0. This means that the function will check the list containing all the agents and added the ones that are in the specified circle into a new ArrayList and return it. Since the neighbor amount determines whether an agent moves or not, it’s really important to implement it correctly and also in an efficient way.

 

Then I run the main function in LandscapeDisplay.java file, which display the landscape with 200 agents.

After that I implements the updatedState() function in SocialAgent class and updateAgent() function in Landscape class to update each agents. If there are many neighbors around a agent then it is not likely to move, otherwise it continues moving randomly until the number of neighbors increases to a certain amount. I tested updateAgent() by creating a Landscape with four agents on it at locations (5, 5), (10, 10), (15, 15), and (20, 20), and the test function correctly shows the number of neighbors.


Then I created SocialAgentSimulation class to run the simulation and the simulation works. The agents moves around and as time goes by many of them get together and stoped.

After that I created CategorizedSocialAgent class and CategorizedSocialAgentSimulation class. CategorizedSocialAgent class extends SocialAgent class and it creates agents with different categories. Besides it overrides updateState() so that it identifies how many neighbors within a radius of 20 have the same category and how many have a different category. If there are many agents with the same category then it tends to stop, otherwise it keeps moving. Then CategorizedSocialAgentSimulation class simulates with agents in two different categories represented by different colors. I conducted three simulation with different radius, 20, 40 or 100, and simulated for 200 times. The results are presented below from left to right. As we can see, the increase of radius leads to less condensed still agents.


Extension

For the first extension, I created DifferentSocialAgentSimulation.java so that user can decide which simulation to run from command line. The default mode is with uncategorized social agent. If the first command line argument is 0, the simulation will use uncategorized agents; if it is 1, the simulation will use categorized agents. Users can also control other parameters, like the size of window and number of agents, from command line arguments.

 

For the second extension, I created AdditionalSocialAgent, which has three categories of agents and the rule changed a little bit. For AdditionalSocialAgent if the number of neighbors is smaller than 2, then the agent would move faster. I also added this agent into DifferentSocialAgentSimulation class so that its simulation can be down with command line first argument as 2.

For the third extension, I did experiments on AdditionalSocialAgent by changing the threshold value of the number of neighbors from 2 to 3 to 4.

After 100 times of moving in each case, the results are shown below.

 

As we can see, as the threshold value increases, there is less chance for the agents to stop.


Reflection

In this project I learned how to create a linked list and how to use it to conduct experiments in agent-based simulation. I also learned how to implement and improve my own data structure, which makes it easier to read and understand. After running the simulation I am clearer with the concept of the Agent-based simulations inspired by Growing Artificial Societies by Epstein and Axtell.
