Di Luo : Di Luo's CS151 Project 9

Created by Di Luo, last modified on Nov 14, 2018

Description

In project 9, I continued to make shape classes but started making use of inheritance and creating a tree class. Inheritance can be used by child classes so that they inherit the methods and fields of the base class. In our project, I created different shape classes, such as Square and Triangle, as the child classes of Shape class. I also modified the L-system class to enable it to use rules that include multiple possible replacement strings and choose them randomly in order to make each tree that I drew unique. Then I created different scenes and even mosaics with the classes.

 

Solutions

First, I made a Tree shape class in tree.py as a child class of Shape class to draw trees, and modified the L-system class to enable it to use rules that include multiple possible replacement strings. Unlike other shape classes, Tree shape class's string is dynamically generated from an L-system so that the shapes of the trees can be different. The L-system object is stored in the lsystem field of the tree class and the number of iterations is stored in another field of tree class. By changing different part of the class and adding a test function, I drew three different trees in the image shown below.

(Required Image 1)

Next, I created other three child classes of Shape class, including Star, Rhombus and Trapezoid, in shapes.py. Each of them can be filled with a color. Then I wrote a test function to draw an image that incorporates all the shapes I created.

(Required Image 2)

Then in home.py I created a new scene with the title "My Home in China" with the shape classes and the tree shape class that I wrote before.

(Required Image 3)

Finally I created the file mosaic.py to draw a mosaic. I wrote two functions, one is tile() for drawing a set of shapes inside a square that is scale by scale in size with the lower left corner of the tile at location (x, y), and the other is mosaic() which draws a 2D array of tiles Nx by Ny. Each tile in the mosaic is of size scale by scale, and the lower left corner of the mosaic is at (x, y). I incorporated four trapezoids with different colors to create the square tile, and then created the 5 x 4 mosaic.

(Required Image 4)


Extensions

For extension, firstly, I created a Ngon class to draw n-gon. The string for drawing in Ngon class is dynamic depending on the number of the sides that imported from the user.

For the second extension, I wrote two new L-systems, rewrote drawString function in turtle_interpreter.py by adding more vocabularies, and drew three trees for each of the new L-systems in home.py. Each of the new trees is unique and on the tree there are a lot of leaves and berries.

For the third extension, I made two more tile functions and mixed them around with the original one in another mosaic function. One of the new tiles includes four rhombi with different colors and directions, and the other one includes a yellow star and a red pentagon inlaid in the middle of star. The red pentagon is created from Ngon class, which is another extension. A new 6 x 6 mosaic is created.

For the fourth extension, I made a hexagon tile, which is combined by six triangles with different colors, and created a function that can draw mosaics incorporated by hexagon tiles. In order to create the mosaic, a more complex double for loops was created and the loop would be different depending on Nx is odd or even.

For the fifth extension, I made an animation of my tiles.

 

Reflection

In project 9, I reviewed the knowledge in turtle, L-system and classes that we learned in class. I modified lsystem.py so that now I can use rules that include multiple possible replacement strings and choose them randomly in order to make unique L-system trees, which is closer to the real world. I learned how to use inheritance between classes to create different child shape classes. In the project only shape classes are used to draw shapes without using any turtle codes directly.
